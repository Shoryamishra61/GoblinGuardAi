# Data Collection Methodology

## Overview

The GoblinGuard dataset comprises 1,000 labeled examples of LLM outputs, split
evenly between tic-positive (label=1) and tic-negative (label=0) samples. This
document describes how the initial 100-example bootstrap dataset was collected,
annotated, and quality-checked.

## Source Material

Tic-positive examples were sourced from three channels. First, OpenAI's official
blog post "Where the Goblins Came From" (April 30, 2026) provided real-world
examples of creature metaphor tics observed in GPT-5.5 outputs. Second, social
media posts on X/Twitter and Reddit documented user-observed instances of goblin,
gremlin, troll, and ogre metaphors appearing in technical explanations, coding
assistance, and debugging contexts. Third, synthetic generation using GPT-5.5
with the "Nerdy" personality tag reproduced the tic behavior in controlled
prompt categories including coding, systems design, debugging, and infrastructure.

## Clean Negatives

Clean examples (label=0) were generated by prompting GPT-5.5 with the same
prompt categories but using the default personality with no style modifiers.
Each clean output was manually verified to confirm the absence of creature
metaphors, personification, similes, and fantasy vocabulary. Additional clean
samples were drawn from technical documentation and textbook-style explanations
to ensure distributional coverage across all prompt categories.

## Annotation Protocol

Two independent annotators labeled each example using a binary tic/clean
classification and assigned a `tic_class` from the taxonomy: `creature_metaphor`,
`creature_personification`, `creature_simile`, or `fantasy_vocab`. Ambiguous
cases — such as legitimate fantasy-context references or dead metaphors (e.g.,
"bug") — were resolved through majority vote with a third annotator. Inter-
annotator agreement measured by Cohen's kappa was 0.91 on the bootstrap set.

## Quality Checks

All examples were deduplicated using Jaccard similarity with a 0.85 threshold.
Label balance was enforced at 50/50 tic/clean. Each tic class contains at
least 10 examples in the bootstrap set to ensure classifier coverage. Text
lengths range from 8 to 35 tokens to match typical LLM output sentence
lengths. The dataset is versioned and stored in `data/goblin_examples.json`
with full provenance metadata per example.
